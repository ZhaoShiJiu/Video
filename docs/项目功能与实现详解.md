## 一、项目定位

MoneyPrinterTurbo 是一个**全自动 AI 短视频生成系统**。你只需要输入一个视频主题（比如"金钱的作用"），系统就能自动完成：

- 文案撰写（LLM）
- 配音（TTS）
- 字幕生成（Whisper / Edge）
- 素材匹配（Pexels / Pixabay / Coverr）
- 视频合成（MoviePy + FFmpeg）

最终输出一个带配音、字幕、背景音乐的高清短视频。

它同时提供三种使用方式：

| 入口 | 启动方式 | 端口 |
|------|---------|------|
| **WebUI** | `streamlit run webui/Main.py` | 8501 |
| **REST API** | `python main.py` | 8080 |
| **CLI** | `python cli.py --video-subject "..."` | — |

## 三、功能实现详解

### 3.1 AI 脚本生成

**文件位置**：`app/services/llm.py` → `generate_script()`（第 631 行）

**功能说明**：用户输入视频主题（如"金钱的作用"），LLM 自动生成一段分段的视频旁白文案。用户也可以手动输入脚本，或者提供额外的提示词来引导 LLM 的写作风格。

**数据流程**：

```
用户输入 → build_script_prompt() 构造 Prompt
        → _generate_response() 调用 LLM
        → format_response() 后处理清洗
        → 输出脚本文本
```

**Prompt 构造**（`build_script_prompt()`，第 595 行）：

使用精心设计的 system prompt（`DEFAULT_SCRIPT_SYSTEM_PROMPT`），核心约束包括：

- 不输出任何 markdown 格式或标题
- 不输出"欢迎观看"等废话，直入主题
- 不输出"旁白：""讲述人："等角色标注
- 不提及 prompt 或脚本本身
- 使用与视频主题相同的语言回复

运行时参数拼接在末尾：

```
# Initialization:
- video subject: {主题}
- number of paragraphs: {段落数}
- language: {语言}

# Additional User Requirements:
{用户自定义提示词}
```

用户可以通过 `custom_system_prompt` 完全替换默认 system prompt，实现高度定制。

**LLM 调用**（`_generate_response()`，第 119 行）：

支持 20+ LLM 提供商，通过 `config.toml` 中的 `llm_provider` 配置项切换。详见 [3.12 多 LLM 提供商支持](#312-多-llm-提供商支持)。

所有提供商共享同一套重试逻辑（`_max_retries = 5`）。

**特殊清洗**（`_normalize_text_response()`，第 41 行）：

```python
# DeepSeek R1 等 reasoning 模型的 <think> 标签自动清除
content = _THINK_BLOCK_RE.sub("", content)          # 完整 <think>...</think>
content = _UNCLOSED_THINK_BLOCK_RE.sub("", content)  # 未闭合的 <think>...
# 去除换行符，保证脚本段落是连续的文本
return content.replace("\n", "")
```

**后处理**（`format_response()`，第 660 行）：

```python
response = response.replace("*", "")      # 去星号
response = response.replace("#", "")      # 去井号
response = re.sub(r"\[.*\]", "", response) # 去 markdown 链接文本
response = re.sub(r"\(.*\)", "", response) # 去 markdown 链接 URL
paragraphs = response.split("\n\n")        # 按双换行分段
```

### 3.2 关键词提取

**文件位置**：`app/services/llm.py` → `generate_terms()`（第 721 行）

**功能说明**：从生成的脚本中提取用于搜索视频素材的英文关键词。例如脚本讲"金钱的起源"，可能提取：

```json
["ancient coins", "barter trade", "gold history", "currency exchange"]
```

**Prompt 构造**：

```
# Role: Video Search Terms Generator

## Goals:
Generate {amount} search terms for stock videos

## Constrains:
1. 返回 JSON 字符串数组
2. 每个搜索词 1-3 个词，必须包含视频主题
3. 只返回 JSON，不返回任何其他内容
4. 搜索词必须与视频主题相关
5. 只用英文回复

## Output Example:
["search term 1", "search term 2", "search term 3", "search term 4", "search term 5"]

## Context:
### Video Subject
{主题}

### Video Script
{脚本}
```

**两种模式**：

| 模式 | 触发条件 | 关键词数量 | 行为 |
|------|---------|:----------:|------|
| 常规模式 | `match_script_order=False` | 5 | 按主题相关度生成 |
| 顺序匹配模式 | `match_script_order=True` | 8 | 按脚本叙事顺序生成 |

**JSON 解析**（第 797 行）：

```python
# 1. 先尝试直接 json.loads()
search_terms = json.loads(_strip_code_fence(response))

# 2. 失败后用正则兜底提取 [...]
match = re.search(r"\[.*]", response, re.DOTALL)
if match:
    search_terms = json.loads(match.group())
```

`_strip_code_fence()` 方法处理 LLM 可能包裹的 markdown 代码块标记（`` ```json ... ``` ``）。

### 3.3 语音合成（TTS）

**文件位置**：`app/services/voice.py` → `tts()`（第 301 行）

**功能说明**：将脚本文字转为 MP3 配音音频。同时生成字幕时间轴信息（哪个词在什么时刻被朗读），供后续字幕生成使用。

**TTS 路由器**：

`tts()` 函数根据 `voice_name` 前缀分发到不同引擎：

```
tts() 路由逻辑：
  ├── "no-voice"     → generate_silent_audio()      用 FFmpeg 生成静音 MP3
  ├── 含 "-V2"       → azure_tts_v2()                Azure 认知服务语音 SDK
  ├── "siliconflow:" → siliconflow_tts()             硅基流动 CosyVoice2
  ├── "gemini:"      → gemini_tts()                  Google Gemini TTS
  ├── "mimo:"        → mimo_tts()                    小米 MiMo V2.5 TTS
  └── 默认           → azure_tts_v1()                Microsoft Edge TTS（免费）
```

#### 3.3.1 Azure TTS V1 — Edge TTS（默认，免费）

**函数**：`azure_tts_v1()`（第 634 行）

基于 `edge_tts` 库，利用微软 Edge 浏览器的免费 TTS 接口。

**核心流程**：

```python
# 1. 兼容新旧版 edge_tts，构造 Communicate 对象
communicate = create_edge_tts_communicate(text, voice_name, rate_str)

# 2. 流式消费音频数据，边生成边写入文件
stream_edge_tts_chunks(communicate, _handle_chunk, timeout_seconds=...)

# 3. 回调函数处理每个 chunk
def _handle_chunk(chunk):
    if chunk["type"] == "audio":
        file.write(chunk["data"])          # 写入音频数据
    elif chunk["type"] in ["WordBoundary", "SentenceBoundary"]:
        sub_maker.feed(chunk)              # 记录逐词时间戳
```

**版本兼容**（`create_edge_tts_communicate()`，第 477 行）：

```python
# 探测 Communicate 构造函数是否接受 boundary 参数
communicate_signature = inspect.signature(edge_tts.Communicate)
if "boundary" in communicate_signature.parameters:
    communicate_kwargs["boundary"] = "WordBoundary"  # 7.x 新版
# 旧版本不传 boundary，保证兼容
```

**流式兼容**（`stream_edge_tts_chunks()`，第 587 行）：

```python
# 新版：同步迭代 stream_sync()
if hasattr(communicate, "stream_sync"):
    for chunk in communicate.stream_sync():
        on_chunk(chunk)

# 旧版：创建事件循环运行异步 stream()
else:
    loop = asyncio.new_event_loop()
    loop.run_until_complete(_consume_async_stream())
```

**超时保护**（`_stream_edge_tts_sync_with_timeout()`，第 534 行）：

核心思路是用 daemon 线程做阻塞 I/O，主线程通过 `queue.Queue.get(timeout=0.5)` 轮询。超时后抛出 `TimeoutError`，让外层重试逻辑继续工作，避免 WebUI 任务**永久卡住**。默认超时 30 秒，可在 `config.toml` 中调整 `edge_tts_timeout`。

**速率转换**（`convert_rate_to_percent()`，第 366 行）：

edge-tts 要求带符号百分比的速率格式。`voice_rate = 1.0` → `"+0%"`，`voice_rate = 0.8` → `"-20%"`。

#### 3.3.2 Azure TTS V2 — Azure 认知服务

**函数**：`azure_tts_v2()`（第 830 行）

使用 Azure 官方 `azure.cognitiveservices.speech` SDK。

**核心差异**：通过事件回调获取精确的逐词时间戳：

```python
def speech_synthesizer_word_boundary_cb(evt):
    duration = _format_duration_to_offset(str(evt.duration))
    offset = _format_duration_to_offset(evt.audio_offset)
    sub_maker.subs.append(evt.text)
    sub_maker.offset.append((offset, offset + duration))

speech_synthesizer.synthesis_word_boundary.connect(
    speech_synthesizer_word_boundary_cb
)
```

需要 Azure 账号的 `speech_key` 和 `speech_region`。

**函数**：`siliconflow_tts()`（第 689 行）

调用硅基流动 REST API：

```python
POST https://api.siliconflow.cn/v1/audio/speech
{
    "model": "FunAudioLLM/CosyVoice2-0.5B",
```

**函数**：`gemini_tts()`（第 925 行）

使用 Google Gemini 2.5 Flash 的 TTS 预览能力：

#### 3.3.6 无配音模式

**函数**：`generate_silent_audio()`（第 251 行）

用 FFmpeg 的 `anullsrc` 滤镜直接生成静音 MP3：

```bash
ffmpeg -y -f lavfi -i "anullsrc=r=44100:cl=mono" -t {duration} -codec:a libmp3lame -q:a 4 {output}
```

**时长估算**（`estimate_no_voice_duration()`，第 214 行）：

```python
CJK字符时长 = cjk_chars / 4.2     # 中文约 4.2 字/秒
英文词时长 = words / 2.7          # 英文约 2.7 词/秒
其他文字时长 = other_chars / 4.0  # 兜底
停顿时长 = (断句数 - 1) * 0.35    # 每句间停顿 0.35 秒
总时长 = 以上四项之和（最少 3 秒）
```

#### 3.3.7 SubMaker 兼容层

**背景**：edge_tts 7.x 的字幕结构从旧版的 `subs/offset` 改为了 `cues`，但项目中 Azure V2、Gemini、SiliconFlow 等非 edge 路径仍然沿用旧结构。为了统一处理，设计了兼容层：

- `ensure_legacy_submaker_fields()`（第 390 行）：为没有 `subs/offset` 的对象补齐空列表
- `populate_legacy_submaker_with_full_text()`（第 405 行）：按标点断句 + 字符数比例分配时间轴，填充 `subs/offset` 结构
- `get_audio_duration()`（第 1426 行）：优先读取 `cues[-1].end`，fallback 到 `offset[-1][1]`

### 3.4 字幕生成

**功能说明**：生成与配音时间轴对齐的 SRT 字幕文件。支持两种 provider，并提供智能校正。

**涉及文件**：

| 文件 | 函数 | 作用 |
|------|------|------|
| `app/services/voice.py` | `create_subtitle()` | Edge 模式字幕生成 |
| `app/services/subtitle.py` | `create()` | Whisper 模式字幕生成 |
| `app/services/subtitle.py` | `correct()` | 字幕与脚本文本校正 |

#### 3.4.1 Edge 模式（默认）

**函数**：`create_subtitle()`（`voice.py` 第 1368 行）

核心思路：将 TTS 返回的逐词时间戳**聚合**为按脚本断句的字幕段落。

**流程**：

```
1. _format_text() 清理脚本文本（去 Markdown、去括号、归一化）
2. split_string_by_punctuations() 将脚本按标点拆为句子列表
3. 判断数据结构：
   - 有 cues  → _build_subtitle_items_from_edge_cues()
   - 有 subs/offset → _build_subtitle_items_from_legacy_submaker()
4. 逐个消费单词/片段，累积文本
5. _match_script_line() 三层匹配：
   ┌─────────────────────────────────────────────────────┐
   │ 第1层: 精确匹配  current_text == script_line        │
   │ 第2层: 去标点匹配  re.sub(r"[_\W]+", "", ...)        │
   │ 第3层: 阿拉伯语归一化  _normalize_arabic()           │
   └─────────────────────────────────────────────────────┘
6. 匹配成功 → 形成 SRT 段落
7. 写入 SRT 文件 + moviepy 可读性验证
```

**SRT 格式**：

```
1
00:00:00,000 --> 00:00:02,500
金钱是一种社会工具

2
00:00:02,500 --> 00:00:05,000
它让交易变得更加高效
```

#### 3.4.2 Whisper 模式

**函数**：`create()`（`subtitle.py` 第 21 行）

使用 `faster-whisper` 模型对音频做语音识别，直接生成带时间戳的字幕。

**模型配置**：

```python
model = WhisperModel(
    model_size_or_path="large-v3",   # 模型大小
    device="cpu",                     # 或 "cuda"
    compute_type="int8",             # 量化精度
)
segments, info = model.transcribe(
    audio_file,
    beam_size=5,
    word_timestamps=True,            # 获取逐词时间戳
    vad_filter=True,                 # 启用 VAD 过滤静音
    vad_parameters=dict(min_silence_duration_ms=500),
)
```

Word 级别的断句策略：检测到包含标点符号的 word 时就切断，形成一个独立字幕段。

**Fallback 机制**：当 Edge 模式生成字幕失败时（文件不存在或为空），自动回退到 Whisper 模式。

#### 3.4.3 字幕校正

**函数**：`correct()`（`subtitle.py` 第 200 行）

解决 Whisper 识别文本与原始脚本不完全一致的问题（尤其对专业术语、人名的识别错误）。

**算法**：

```python
while script_index < len(script_lines) and subtitle_index < len(subtitle_items):
    if script_line == subtitle_line:
        # 完全一致，保留
        new_subtitle_items.append(subtitle_items[subtitle_index])
    else:
        # 不一致：尝试合并连续字幕段
        # 用 Levenshtein 编辑距离计算相似度
        if similarity(script_line, combined_subtitle) > 0.8:
            # 相似度 > 80%：用脚本原文替换字幕文字，保留时间戳
            new_subtitle_items.append((index, time_range, script_line))
        else:
            # 兜底：也用脚本原文（宁可字幕文字准确）
            new_subtitle_items.append((index, time_range, script_line))
```

**Levenshtein 编辑距离实现**（`subtitle.py` 第 174 行）：标准动态规划算法，不用第三方库。

### 3.5 素材搜索与下载

**文件位置**：`app/services/material.py`

**功能说明**：根据关键词从免版权素材库搜索视频，下载到本地缓存目录。

#### 3.5.1 三个素材源

| 素材源 | 搜索函数 | API 端点 | 特点 |
|--------|---------|---------|------|
| **Pexels** | `search_videos_pexels()` | `api.pexels.com/videos/search` | 支持横竖屏过滤，质量稳定 |
| **Pixabay** | `search_videos_pixabay()` | `pixabay.com/api/videos/` | 免费，按宽度过滤 |
| **Coverr** | `search_videos_coverr()` | `api.coverr.co/videos` | 免费高清/4K，以横屏为主 |

**统一搜索模式**：

```python
def search_videos_xxx(search_term, minimum_duration, video_aspect):
    # 1. 构造 HTTP 请求（Bearer Token 认证）
    # 2. 发送 GET 请求（30s 连接超时 + 60s 读取超时）
    # 3. 解析 JSON 响应
    # 4. 过滤条件：
    #    - duration >= minimum_duration（确保素材够长）
    #    - width × height 匹配目标分辨率（避免模糊拉伸）
    # 5. 返回 List[MaterialInfo]
```

**Coverr 特别说明**：Coverr 的 duration 字段可能是 `number` 或 `string` 类型，代码中做了兼容：

```python
duration = int(float(v.get("duration") or 0))
```

Coverr 库以 16:9 横屏为主，竖屏占比极低（约 1%），因此不做 aspect_ratio 过滤，由下游的 resize + letterbox 逻辑统一处理。

#### 3.5.2 素材下载与缓存

**函数**：`save_video()`（第 244 行）

#### 3.5.3 下载调度策略

**函数**：`download_videos()`（第 304 行）

**普通模式**（`match_script_order=False`）：

```
所有关键词遍历搜索 → 去重（按 URL） → 统计总时长
  → random 模式：随机打乱候选列表
  → sequential 模式：保持 API 返回顺序
  → 逐个下载，累计时长达到音频时长后停止
```

**顺序匹配模式**（`_download_videos_by_script_order()`，第 386 行）：

核心思路是按关键词**分组后轮询下载**：

```
关键词1: [视频A1, 视频A2, 视频A3, ...]
关键词2: [视频B1, 视频B2, ...]
关键词3: [视频C1, 视频C2, 视频C3, ...]

下载顺序: A1 → B1 → C1 → A2 → B2 → C2 → ...
         ↑第1轮轮询↑   ↑第2轮轮询↑

这样确保画面顺序贴近文案叙事顺序，
避免第一个关键词的视频把后续主题排挤出时间线。
```

**函数**：`combine_videos()`（第 500 行）

**步骤 1 — 素材切分与去重**：

**字幕视觉居中**（`_get_visible_center_position()`，第 813 行）：

```python

visible_bottom = ys.max()
visible_height = visible_bottom - visible_top + 1
y = int((container_height - visible_height) / 2 - visible_top)
```

```python
    # 返回 206 Partial Content + StreamingResponse
```

#### 安全设计

- **路径穿越防护**：所有文件操作通过 `file_security.resolve_path_within_directory()` 限制在任务目录或白名单目录内
- **上传文件名清理**：`_sanitize_upload_filename()` 剥离目录信息，只保留纯文件名
- **Token 认证**：框架已存在（`base.verify_token`），但默认关闭

### 4.3 容错设计

| 场景 | 处理方式 |
|------|---------|
| LLM 请求失败 | 最多 5 次重试 |
| TTS 生成失败 | 最多 3 次重试；0 字节文件自动清理 |
| Edge 字幕失败 | 自动 fallback 到 Whisper 模式 |
| Whisper 识别错误 | Levenshtein 编辑距离校正（相似度 > 80% 则采纳脚本原文） |
| 硬件编码失败 | 自动回退 libx264；进程内禁用该编码器 |
| 损坏图片文件 | 自动生成剥离元数据的 PNG 副本 |
| 路径穿越攻击 | `file_security.resolve_path_within_directory()` 防护所有文件操作 |
| 字幕匹配失败 | 三层匹配（精确 → 去标点 → 阿拉伯语归一化） |

### 4.4 兼容性设计

| 场景 | 实现方式 |
|------|---------|
| edge_tts 新旧版本 | `inspect.signature()` 探测参数支持 |
| SubMaker 新旧结构 | `ensure_legacy_submaker_fields()` 补齐 |
| 流式消费同步/异步 | `stream_edge_tts_chunks()` 统一处理 |
| 容器/本机环境 | `is_running_in_container()` 自适应 |
| voice_name 多格式 | `parse_voice_name()` 统一解析 |
| Gemini 旧模型名 | `_DEPRECATED_GEMINI_MODELS` 自动升级 |
| Coverr duration 类型 | `int(float(...))` 兼容 number/string |
| Windows/Mac/Linux 路径 | `_format_ffmpeg_concat_path()` 统一处理 |
